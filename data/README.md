# Data

The demo runs on synthetic data generated by `src/data_sim.py`, so nothing needs
to be downloaded to try the app. To move to real biology, drop the organiser
dataset (or a BV-BRC pull) in here and point `train.load_data()` at it.

## Expected format

Two tables, joined on `genome_id`.

**features.csv** — one row per genome, one 0/1 column per resistance gene or
mutation (from AMRFinderPlus), plus one `target_<Antibiotic>` column per drug.

| genome_id | blaKPC-2 | blaCTX-M-15 | ... | target_Meropenem |
|-----------|----------|-------------|-----|------------------|
| GEN_00001 | 1        | 0           | ... | 1                |

**labels.csv** — one row per genome, one column per antibiotic. `1` = resistant
(likely to fail), `0` = susceptible (likely to work), blank = not tested. Use only
laboratory-measured results, not model-predicted phenotype fields.

**groups** — a `lineage_group` column (or a separate file) assigning each genome
to a genetically related cluster, so training can hold out whole lineages.

## Real source (mentor-confirmed recipe)

Three places to look, in order of usefulness for this project.

1. **BV-BRC** (bv-brc.org), formerly PATRIC. The main source. Search your pathogen,
   open **Genomes**, and download assembled genomes (limit 10,000 per pull, one pull
   is plenty). The genome table also gives the **AMR phenotype per strain**
   (strain, antibiotic, resistance phenotype), which are your labels, and it can
   generate a phylogenetic tree to inform the grouped split.
2. **NCBI Bacterial AMR Reference Gene Database** and **MicroBIGG-E**. Small, curated
   sets of the resistance genes themselves. Filter by organism, then Send to →
   Coding sequences → FASTA nucleotide/protein → Create file. This is the curated
   reference AMRFinderPlus screens against.
3. **Plain NCBI genome search** if you ever need more genomes. Larger, but AMR
   annotation completeness drops as the set grows.

### How much
- **100 genomes** is enough to prototype locally, **~1,000** makes a solid dataset.
- **One species, one disease.** Use laboratory-measured phenotypes as labels.
- Avoid bias: do not take the first N alphabetically, and check the subset is not
  all the same phenotype. Spread across isolates and collection years.
- Group genomes by genetic similarity for the split. Keep a fixed, HIDDEN test set.
